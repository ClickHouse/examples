# What is NDJSON / JSON Lines?

Runnable companion to the article
[What is NDJSON / JSON Lines?](https://clickhouse.com/resources/engineering/what-is-ndjson).

Generates a small NDJSON file (one JSON object per line) and the same rows as a
single JSON array, then uses `clickhouse local` to infer the schema, query both,
and prove that NDJSON streams line-by-line.

## Run it

```bash
./generate.sh   # writes data/events.ndjson, data/events_array.json, data/events_large.ndjson
./run.sh        # DESCRIBEs + queries the NDJSON, reads the array, streams the large file
```

Requirements: `clickhouse` (install with `clickhousectl`: `curl https://clickhouse.com/cli | sh` then `clickhousectl local use latest`), invoked as `clickhouse local`. Python 3 (stdlib only) is used once in `generate.sh` to build the array document.

Override row counts for a fast verify:

```bash
SMALL_ROWS=8 LARGE_ROWS=200000 ./generate.sh
```

## What's in the files

Synthetic event rows: `id`, `event_time`, `country`, `event_type`, `revenue`.

- `events.ndjson` — `FORMAT JSONEachRow`, one object per line.
- `events_array.json` — the same shape wrapped in a single `[ ... ]` array.
- `events_large.ndjson` — 1,500,000 rows (~147 MB) for a perf number.

## Notes on reproducibility

`revenue` uses `randUniform`, so the exact values (and therefore the `sum(revenue)`
figures) differ run-to-run, and the small NDJSON and array files are generated by
separate calls so their revenue values differ from each other. Schema, row counts,
and country distribution are stable. `expected_output.txt` is a real capture from
one run.
